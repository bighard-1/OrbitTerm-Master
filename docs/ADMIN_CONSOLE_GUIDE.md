# OrbitTerm 管理端小白操作指南

本文面向没有后端经验的管理员，说明如何使用内置管理端完成用户治理、策略配置、备份自检与诊断导出。

## 1. 入口

部署完成后访问：

```text
https://你的后端域名/admin-console/
```

管理端复用后端 `/api/v1/admin/...` 接口，不会额外保存密码、Token 或用户主密码。

## 2. 首次初始化

1. 在服务器环境变量中临时配置高强度 `ADMIN_BOOTSTRAP_TOKEN`。
2. 打开管理端页面，输入初始化令牌、首个管理员账号和密码。
3. 创建完成后立即登录。
4. 如果首个管理员已创建，建议清空或轮换 `ADMIN_BOOTSTRAP_TOKEN` 后重启后端容器。

## 3. 用户治理

管理端支持：

- 按用户名、状态、角色筛选用户，并通过分页处理较大规模账号列表。
- 创建受管用户或管理员账号。
- 调整角色，角色变更后目标用户旧 Token 会失效。
- 按分钟设置封禁周期，到期后可由后台任务自动解封。
- 手动解封、强制下线、重置登录密码。
- 普通用户全部下线，用于 Token 泄漏或密钥轮换后的应急处理；管理员账号不受影响。
- 软删除/注销与恢复用户。
- 扫描已到期封禁。

高危操作需要填写原因，部分操作还需要输入 `CONFIRM`，所有结果都会写入审计日志。

## 4. 主密码边界

管理员只能重置“登录密码”，不能找回或修改用户“主密码”。

原因：OrbitTerm 使用零知识加密，云端只保存客户端加密后的密文资产。后端没有主密码、派生密钥或资产明文。

用户忘记主密码时：

- 如果本地仍有可解密材料，客户端可引导用户迁移或重设。
- 如果没有旧主密码或本地可用解密材料，旧加密资产无法由服务端恢复。

## 5. 策略配置

“系统策略”页面可配置：

- 是否允许新用户注册。
- 最小登录密码长度。
- 注册关闭原因。
- 恢复提示文案和支持联系方式。

这些策略用于提升小白用户的可理解性，同时确保恢复边界足够清晰。

## 6. 运行健康

仪表盘会显示：

- 运行健康：后端和数据库是否连通。
- 自动解封：封禁到期自动解封任务是否启用、扫描间隔和批量上限。
- Token 策略：Access/Refresh 周期和 JWT 密钥强度状态。

如果运行健康显示异常，先检查 PostgreSQL 容器、`DATABASE_URL` 和 1Panel 容器网络。

## 7. 备份自检

“备份自检”页面用于确认：

- PostgreSQL 数据库是否可读。
- 关键表数量是否能统计。
- `JWT_SECRET`、`DATABASE_URL`、`ADMIN_BOOTSTRAP_TOKEN` 等关键变量是否已配置或存在风险。
- 1Panel 部署环境需要备份哪些内容。

注意：页面只展示脱敏值，不显示密钥原文。

## 8. 诊断包导出

在“备份自检”页面点击“导出诊断包”。

诊断包包含：

- Go 运行时、系统架构与 Gin 模式。
- 备份自检报告。
- 最近 20 条审计日志摘要。
- 脱敏策略说明和运维建议。

诊断包不包含：

- `JWT_SECRET` 明文。
- 数据库密码明文。
- `ADMIN_BOOTSTRAP_TOKEN` 明文。
- 用户主密码、服务器密码、私钥、加密资产明文。

## 9. 审计日志

“审计日志”页面可查看管理员敏感操作记录，并支持按 action 或目标用户筛选。需要提交给开发者定位问题时，可导出 JSON。系统策略页可设置审计日志保留天数和单次清理批量；审计页可执行“清理过期”，清理动作也会被记录。

## 10. 建议的日常运维顺序

1. 每次升级前，在 1Panel 中备份 PostgreSQL。
2. 进入管理端执行备份自检。
3. 若出现警告，先处理环境变量或数据库问题。
4. 升级后导出诊断包留档。
5. 定期检查审计日志，确认高危操作均有原因记录。
6. 按合规要求设置审计保留周期，并定期清理过期日志，避免数据库无限增长。
